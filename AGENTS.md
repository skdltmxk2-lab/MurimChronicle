# Project Instructions

## Deployment

- After every user-requested code or content change, run the relevant verification and deploy it to production before reporting completion, unless the user explicitly says not to deploy.
- Deploy this repository by committing only the task-related files to `master` and pushing `origin master`; the GitHub push triggers the Vercel Production deployment.
- Confirm that the GitHub deployment status reaches `success` before reporting the work as complete.
- Never include unrelated dirty-worktree files in a task commit or deployment.
